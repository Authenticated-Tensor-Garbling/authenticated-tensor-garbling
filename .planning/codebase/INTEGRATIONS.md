## Randomness Sources

**CSPRNG:**
- `rand::rng()` — thread-local OS-seeded CSPRNG (wraps platform entropy on all targets)
- `rand_chacha::ChaCha12Rng::seed_from_u64(...)` — deterministic seeded RNG used for ideal functionalities and preprocessing internals; seeds are hardcoded constants (e.g., 42, 43, 44, 45) for reproducibility in tests and benchmarks

**Usage sites:**
- `src/delta.rs` — `Delta::random(rng)` samples global correlation keys
- `src/block.rs` — `Block::random(rng)`, `Block::random_vec(rng, n)`
- `src/bcot.rs` — `IdealBCot::new(seed_a, seed_b)` uses `ChaCha12Rng`
- `src/preprocessing.rs` — four dedicated `ChaCha12Rng` instances with fixed seeds for alpha, beta, correlated, and gamma IT-MAC generation
- `benches/benchmarks.rs` — `setup_auth_gen/eval/pair` use `TensorFpre::new(seed, ...)` which passes seed to `ChaCha12Rng`

## Cryptographic Primitives

**Fixed-Key AES-128 (`src/aes.rs`):**
- `aes::Aes128Enc` via `cipher::{BlockCipherEncrypt, KeyInit}` traits
- Global singleton: `static FIXED_KEY_AES: Lazy<FixedKeyAes>` (key = `[69, 42, 69, 42, ...]`)
- Provides three hash constructions:
  - `tccr(tweak, block)` — Tweakable Circular Correlation-Robust: `π(π(x) ⊕ i) ⊕ π(x)`
  - `tccr_many<N>(tweaks, blocks)` — batched TCCR for GGM tree expansion
  - `cr(block)` — Correlation-Robust: `π(x) ⊕ x`; used in `hash_check_zero` (`src/online.rs:92-98`)
- Reference: Guo et al., eprint 2019/074 §7.2 and §7.4
- ARMv8 AES hardware path active by default on `aarch64-apple-darwin`

**IT-MAC Construction:**
- Information-theoretic MACs via `Key::auth(bit, delta)` — implemented in `src/keys.rs`
- No external MAC library; pure arithmetic over `Block` (XOR of 128-bit values)

**GGM Pseudorandom Tree:**
- GGM seed expansion via `FIXED_KEY_AES.tccr_many(...)` — self-contained, no external PRG library

**blake3 (1.8.2):**
- Listed in `[dependencies]` in `Cargo.toml` but **never imported in `src/` or `benches/`**
- All blake3 usage in the repo is inside `references/mpz-dev/` (a separate reference codebase, not part of this crate's compilation unit)
- This dependency can be removed without any source changes

## Network Simulation

**Simulated Network (not real TCP/QUIC):**
- `benches/network_simulator.rs` — `SimpleNetworkSimulator` struct uses `tokio::time::sleep` to simulate deterministic transit delay
- Parameters: 100 Mbps bandwidth, configurable latency (set to 0 ms in benchmarks)
- Used only in `network_benches` group in `benches/benchmarks.rs` (currently disabled in `criterion_main!`)
- The active `online_benches` group uses a purely arithmetic transit model: `transit_ns(bytes) = bytes * 8 * 1e9 / NETWORK_BANDWIDTH_BPS` added to measured wall-clock time — no tokio, no scheduler jitter

**No real networking:** There is no TCP, TLS, QUIC, or any socket-level communication. The protocol is simulated in-process with both parties' state held in the same process.
