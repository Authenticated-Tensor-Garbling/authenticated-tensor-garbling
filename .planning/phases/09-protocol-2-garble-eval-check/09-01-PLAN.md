```rust
pub fn gen_auth_bit(&mut self, x: bool) -> AuthBit {
    let a = self.rng.random_bool(0.5);   // gen-side value
    let b = x ^ a;                       // eval-side value
    let a_share = build_share(&mut self.rng, a, &self.delta_b);  // committed under delta_b (D_ev)
    let b_share = build_share(&mut self.rng, b, &self.delta_a);  // committed under delta_a (D_gb)
    AuthBit {
        gen_share: AuthBitShare { key: b_share.key, mac: a_share.mac, value: a },
        eval_share: AuthBitShare { key: a_share.key, mac: b_share.mac, value: b },
    }
}
```
**Key property:** `gen_auth_bit()` ALREADY produces a share pair that is simultaneously D_gb-valid (gen.key authenticates eval.value under delta_a) AND D_ev-valid (eval.key authenticates gen.value under delta_b). The same `gen_auth_bit` call is correct for both `*_auth_bit_shares` (D_gb path) and `*_d_ev_shares` (D_ev path); only the contents differ because each field draws an independent random bit.
