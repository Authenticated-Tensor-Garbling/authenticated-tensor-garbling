<paper_formulas>
<!-- D-09 c_gamma per gate, expanded for assembly -->

c_gamma[(i,j)] = (L_alpha[i]    XOR l_alpha[i])
              ⊗ (L_beta [j]    XOR l_beta [j])
              XOR (L_gamma[(i,j)] XOR l_gamma[(i,j)])

Expanding the AND:
  = L_alpha[i] · L_beta[j]                  (PUBLIC ⊗ PUBLIC bit)
    XOR L_alpha[i] · l_beta[j]              (PUBLIC ⊗ shared)
    XOR l_alpha[i] · L_beta[j]              (shared ⊗ PUBLIC)
    XOR l_alpha[i] · l_beta[j]              (shared ⊗ shared = l_gamma*[(i,j)])
    XOR L_gamma[(i,j)]                      (PUBLIC bit)
    XOR l_gamma[(i,j)]                      (shared)

For honest parties this evaluates to v_alpha ⊗ v_beta XOR v_gamma == 0.

Per-gate D_ev-authenticated SHARE assembly (one party at a time), keeping value/key/mac consistent:
  share_per_party = (L_alpha[i] AND L_beta[j])-as-public-bit            // contributes to value only
                  + L_alpha[i] · beta_auth_bit_shares[j]                 // PUBLIC * shared = include if L_alpha[i]
                  + L_beta [j] · alpha_auth_bit_shares[i]                // include if L_beta[j]
                  + correlated_auth_bit_shares[j*n+i]                    // l_gamma*
                  + L_gamma[(i,j)]-as-public-bit                          // contributes to value only
                  + gamma_auth_bit_shares[j*n+i]                          // l_gamma

For the in-process simulation per RESEARCH Open Question 1 recommendation (a) and Pitfall 5 mitigation:
We can sidestep the public-bit MAC subtleties by:
  - Computing the value contribution of public terms (L_alpha[i] AND L_beta[j]) XOR L_gamma[(i,j)] as a bool.
  - Folding that bool into the assembled share's `value` field AFTER the per-party XOR + cross-party XOR.
  - The MAC fields will then satisfy mac == key XOR value*delta_ev because the public bits do not perturb
    the key or mac (they are public — committed with Mac::PUBLIC[bit] and a zero key, which XOR to nothing
    on both sides since both parties hold the same Mac::PUBLIC and Block::ZERO key — so the GF(2)-linear
    IT-MAC invariant holds for the combined share if we XOR the public-bit `value` contribution into the
    pre-XORed share's value field).

Honest-run check: after the assembly, every entry's `value` should be 0 and `mac == key.auth(0, delta_a)`
                  (which simplifies to `mac == key`).

Tampered run (P1-05): if we flip lambda_gb[k] for one k, the evaluator's compute_lambda_gamma produces
                      a wrong L_gamma at index k → that entry's `value` in c_gamma becomes 1 →
                      check_zero returns false.
</paper_formulas>

The cleanest in-process implementation, exploiting that we have BOTH parties' shares in the same test
and that public bits' MACs cancel because both parties contribute identical Mac::PUBLIC values:

1. Pick small n, m (e.g., n=4, m=3 — keeps tests fast).
2. Pick concrete `input_x: usize`, `input_y: usize` bit patterns.
3. Build via `IdealPreprocessingBackend.run(n, m, 1, 1)`.
4. Compute clear `l_alpha[i] = gen.alpha_auth_bit_shares[i].value XOR ev.alpha_auth_bit_shares[i].value`,
   similarly for `l_beta`, `l_gamma`, `l_gamma*`.
5. Compute `L_alpha[i] = ((input_x >> i) & 1 != 0) XOR l_alpha[i]`, similarly for `L_beta[j]`.
   These are the masked input wire values that the garbler reveals.

   IMPORTANT: TensorFpre's `generate_for_ideal_trusted_dealer(input_x, input_y)` is the established
   path that wires input bits into alpha shares (see src/lib.rs:264-265). For
   IdealPreprocessingBackend.run() the alpha/beta authenticated bits are sampled INDEPENDENTLY of any
   user input (the backend calls generate_for_ideal_trusted_dealer(0, 0) — see src/preprocessing.rs:138).
   So in this test we treat the test-defined `input_x`/`input_y` as INDEPENDENT inputs and compute
   `L_alpha[i] = ((input_x >> i) & 1 != 0) XOR l_alpha[i]` ourselves.

   For the L values to be consistent with what `garble_final` actually outputs, the test must NOT
   re-run the garble pipeline against `input_x`/`input_y` — it must use the alpha/beta values that
   IdealPreprocessingBackend produced. The wire-input-to-pipeline flow is independent of the
   correctness check we are writing.

   NOTE: `compute_lambda_gamma` does NOT depend on any specific input wire values — it only uses
   first_half_out (which is whatever garble_final left) and gamma_auth_bit_shares. So the round-trip
   reconstruction `lambda_combined = ev.compute_lambda_gamma(&gb.compute_lambda_gamma())` should
   yield `L_gamma[(i,j)] := v_gamma[(i,j)] XOR l_gamma[(i,j)]` where v_gamma is whatever the
   garble pipeline produced for the inputs that IdealPreprocessingBackend provided.

   For the simplest test, we DO NOT need to assert L_gamma equals "input_x ⊗ input_y XOR l_gamma" —
   the IdealPreprocessingBackend doesn't take user inputs, so v_gamma is determined by its internal
   alpha/beta values. INSTEAD, we assert the consistency check:
     L_alpha ⊗ L_beta XOR L_gamma == l_alpha ⊗ l_beta XOR l_gamma  (i.e., c_gamma = 0)
   where:
     L_alpha[i] = (alpha[i] from preprocessing) XOR (alpha[i] share reconstructed) = the masked alpha
                  Actually L_alpha[i] = lambda_alpha (the masked input) — for IdealPreprocessingBackend
                  there is no meaningful "input"; alpha IS the only data. So we set:
                    L_alpha[i] := alpha_clear[i]  (defined as gen.alpha_auth_bit_shares[i].value
                                                   XOR ev.alpha_auth_bit_shares[i].value, the
                                                   reconstructed l_alpha bit).
                  This means L_alpha XOR l_alpha == 0 trivially → v_alpha == 0 → c_gamma == 0
                  for honest parties (since c_gamma = v_alpha ⊗ v_beta XOR v_gamma = 0 ⊗ 0 XOR v_gamma).

   That degenerate "v_alpha == 0" makes the c_gamma test less exercised. To get a meaningful test,
   we set L_alpha[i] and L_beta[j] to test-chosen bool patterns (e.g., from input_x/input_y) and
   then construct the c_gamma assembly using whatever those L values are. The honest-party identity
   v_gamma = v_alpha ⊗ v_beta still holds because:
     v_gamma[(i,j)] = first_half_out[(i,j)]_gen.lsb() XOR first_half_out[(i,j)]_ev.lsb()  (after garble_final)
   and the existing test_auth_tensor_product proves this matches alpha ⊗ beta. After we pick L_alpha,
   L_beta freely:
     v_alpha[i] := L_alpha[i] XOR l_alpha[i]
     v_beta[j]  := L_beta[j]  XOR l_beta[j]
     v_gamma[(i,j)] := lambda_combined[(j*n+i)] XOR l_gamma[(j*n+i)]
   And we verify v_alpha ⊗ v_beta XOR v_gamma == 0 by construction (the garble pipeline guarantees
   v_gamma == v_alpha ⊗ v_beta when v_alpha and v_beta are the actual masked input bits used by
   garble_first_half / garble_second_half).

   BUT — the IdealPreprocessingBackend's get_clear_values are not exposed by the backend (only by

6. For P1-05: clone `lambda_gb`, flip bit at index 0, then call `ev.compute_lambda_gamma(&tampered)`.

    ```rust
        /// Assembles the per-gate D_ev-authenticated `c_gamma_shares` vec for Protocol 1
        /// CheckZero per Construction 3 (5_online.tex line 206) and Phase 8 CONTEXT.md D-09.
        ///
        /// For each (i, j) gate, c_gamma is the linear combination:
        ///   c_gamma[(i,j)] = (L_alpha[i] AND L_beta[j])              [public bit]
        ///                  XOR L_alpha[i] · l_beta[j]                 [shared, include iff L_alpha[i]]
        ///                  XOR L_beta [j] · l_alpha[i]                [shared, include iff L_beta[j]]
        ///                  XOR l_gamma_star[(i,j)]                    [shared, always — = l_alpha · l_beta]
        ///                  XOR L_gamma[(i,j)]                          [public bit]
        ///                  XOR l_gamma[(i,j)]                          [shared, always]
        ///
        /// Per the caller-pre-XOR contract of `check_zero`, this helper:
        ///   1. Builds each party's per-gate D_ev share by XOR-summing the contributing
        ///      AuthBitShares using the `+` operator (field-wise XOR — see src/sharing.rs:66).
        ///   2. XORs the gen and eval per-gate shares pairwise to produce a single combined
        ///      AuthBitShare whose `value` is the reconstructed c_gamma bit and whose
        ///      mac/key satisfy the IT-MAC invariant under delta_a.
        ///   3. Folds the public-bit value contribution `(L_alpha[i] AND L_beta[j]) XOR L_gamma[(i,j)]`
        ///      into the combined share's `value` field. Public bits contribute identically on
        ///      both parties (Mac::PUBLIC + zero key are deterministic constants), so they
        ///      cancel in the cross-party XOR — folding only the value preserves the
        ///      IT-MAC invariant on the combined share.
        ///
        /// Returns a Vec<AuthBitShare> of length n*m in column-major order (j*n + i).
        #[allow(clippy::too_many_arguments)]
        fn assemble_c_gamma_shares(
            n: usize,
            m: usize,
            l_alpha_pub: &[bool],          // length n — public masked alpha bits
            l_beta_pub: &[bool],           // length m — public masked beta bits
            l_gamma_pub: &[bool],          // length n*m — public masked gamma bits (column-major)
            gb: &AuthTensorGen,
            ev: &AuthTensorEval,
        ) -> Vec<AuthBitShare> {
            assert_eq!(l_alpha_pub.len(), n);
            assert_eq!(l_beta_pub.len(),  m);
            assert_eq!(l_gamma_pub.len(), n * m);
            assert_eq!(gb.alpha_auth_bit_shares.len(),       n);
            assert_eq!(gb.beta_auth_bit_shares.len(),        m);
            assert_eq!(gb.correlated_auth_bit_shares.len(),  n * m);
            assert_eq!(gb.gamma_auth_bit_shares.len(),       n * m);
            assert_eq!(ev.alpha_auth_bit_shares.len(),       n);
            assert_eq!(ev.beta_auth_bit_shares.len(),        m);
            assert_eq!(ev.correlated_auth_bit_shares.len(),  n * m);
            assert_eq!(ev.gamma_auth_bit_shares.len(),       n * m);

            let mut out: Vec<AuthBitShare> = Vec::with_capacity(n * m);
            for j in 0..m {
                for i in 0..n {
                    let idx = j * n + i;

                    // Per-party D_ev share contributions for the SHARED terms.
                    // Each starts at AuthBitShare::default() (key=0, mac=0, value=0).
                    let mut gb_share = AuthBitShare::default();
                    let mut ev_share = AuthBitShare::default();

                    // Term: L_alpha[i] · l_beta[j]   (include iff L_alpha[i] is true)
                    if l_alpha_pub[i] {
                        gb_share = gb_share + gb.beta_auth_bit_shares[j];
                        ev_share = ev_share + ev.beta_auth_bit_shares[j];
                    }

                    // Term: L_beta[j] · l_alpha[i]   (include iff L_beta[j] is true)
                    if l_beta_pub[j] {
                        gb_share = gb_share + gb.alpha_auth_bit_shares[i];
                        ev_share = ev_share + ev.alpha_auth_bit_shares[i];
                    }

                    // Term: l_alpha[i] · l_beta[j] = l_gamma*[(i,j)]   (always)
                    gb_share = gb_share + gb.correlated_auth_bit_shares[idx];
                    ev_share = ev_share + ev.correlated_auth_bit_shares[idx];

                    // Term: l_gamma[(i,j)]   (always)
                    gb_share = gb_share + gb.gamma_auth_bit_shares[idx];
                    ev_share = ev_share + ev.gamma_auth_bit_shares[idx];

                    // Cross-party pairwise XOR — combined.value is now the reconstructed
                    // bit of (the shared part of c_gamma); combined.mac == combined.key
                    // when value is 0 (IT-MAC invariant under delta_a).
                    let mut combined = gb_share + ev_share;

                    // Fold the PUBLIC-bit value contribution.
                    // The public terms are (L_alpha[i] AND L_beta[j]) and L_gamma[(i,j)].
                    // These contribute only to value (each party would XOR identical
                    // Mac::PUBLIC[bit] and zero key — they cancel in the cross-party XOR).
                    let public_bit = (l_alpha_pub[i] & l_beta_pub[j]) ^ l_gamma_pub[idx];
                    combined.value ^= public_bit;

                    out.push(combined);
                }
            }
            out
        }
    ```
