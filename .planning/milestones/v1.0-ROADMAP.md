## Milestone Summary

**Key Decisions:**

- Implement Pi_LeakyTensor via GGM tree macro — paper spec; direct-AND approach was not the protocol
- In-process F_eq (ideal) — matches IdealBCot pattern; no networking needed
- Pi_aTensor' (permutation bucketing) over Pi_aTensor — better bucket size (log(nℓ) vs log(ℓ))
- Keep TensorFpreGen/Eval interface — online phase already correct; minimize scope
- tensor_macro as standalone pub(crate) module — no dependency on leaky_tensor_pre or preprocessing
- Hard-code shuffle_seed=42 in run_preprocessing — deterministic permutations for test reproducibility
- Construction 4 replaces Construction 3 fully — no parallel path; single bucket-sizer

**Issues Resolved:**

- Bug 1 (GGM tree macro absent): Implemented tensor_garbler/tensor_evaluator in Phase 3
- Bug 2 (F_eq missing): Implemented feq::check in Phase 4
- Bug 3 (wrong combining algorithm): Implemented two_to_one_combine in Phase 5
- Bug 4 (gamma bits in wrong place): Removed from LeakyTriple in Phase 4
- Bug 5 (wrong bucket size formula): Fixed bucket_size_for(n, ell) in Phase 6
- Bug 6 (wire labels in wrong layer): Removed from LeakyTriple in Phase 4
- Bug 7 (generate_with_input_values violates input independence): Renamed + documented in Phase 2
- Bug 8 (tests echo code not paper): Replaced with paper-invariant tests in Phases 3–6
- COT choice-bit convention (fix-cot-convention quick task): Resolved by Phase 4 rewrite applying verifier-delta convention correctly from scratch

**Issues Deferred:**

- Real OT protocol (Ferret/IKNP) replacing ideal F_bCOT — v2
- Network communication layer — v2
- Malicious security simulation proof — v2

**Technical Debt Incurred:**

- shuffle_seed=42 hardcoded in run_preprocessing — caller should supply seed for production use
- IdealBCot stays as the correlated randomness source — real OT deferred
